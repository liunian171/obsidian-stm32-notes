# 端到端流程与架构设计哲学

## 6阶段流程

```
数据采集 → 数据清洗 → 模型训练 → 模型导出 → Jetson部署 → MCU执行
```

### 阶段1: 数据采集
```
操作者 ←遥控器→ MCU(STM32)
                        │
              UART 50Hz 控制数据
                        ▼
              Jetson Orin Nano
                        │
              CSI摄像头 30fps
                        ▼
              同步保存: 图像 + 控制值
              输出: raw_data/run_20260301_120000/
                    ├── frames/000000.jpg
                    ├── controls.csv
                    └── meta.json
```

### 阶段2: 数据清洗
```
输入: 原始图像+控制配对
处理:
  1. 剔除模糊/异常帧
  2. 时间对齐校验
  3. 标注target_valid (有人/无人)
  4. 整理成标准目录结构
输出: dataset/0312/{main1, main2, no_target}/
```

### 阶段3: 模型训练
```
输入: 清洗后数据集 (5帧图像 + 5个速度 + 4维标签)
模型: ResNet18 + GRU (端到端行为克隆)
Loss: 加权多任务 (Steer SmoothL1 + Thr SmoothL1 + Brake BCE + TV BCE)
输出: checkpoints/20260301_120000/best_model.pth
```

### 阶段4: 模型导出
```
输入: best_model.pth (PyTorch checkpoint)
处理: 加载权重, 设eval模式, pretrained_backbone=False
输出: 可部署模型 (当前为PyTorch原生格式)
```

### 阶段5: Jetson部署
```
输入: 模型权重 + CSI摄像头实时流
启动:
  python3 main.py --config config/autodrive.yaml
处理:
  1. 三线程: 摄像头/推理/发送
  2. 推理频率10Hz, 发送频率20Hz
  3. 安全后处理: 滤波/死区/刹车迟滞/target_valid
输出: UART控制指令 (TYPE=0x10, 20Hz)
```

### 阶段6: MCU执行
```
输入: UART接收22字节帧
处理:
  1. CRC校验
  2. 解析steer/throttle/brake
  3. 模式=FOLLOW, 执行Jetson指令
输出:
  CAN → 转向伺服 (绝对位置)
  TIM3 PWM → 轮毂电机
  GPIO → 刹车电缸
```

## 架构设计哲学

### 1. 为什么拆成6个子项目?

| 原因 | 说明 |
|------|------|
| 工具链不同 | MCU: GCC+STM32CubeIDE / Jetson: Python / 训练: PyTorch / 机械: CAD |
| 部署目标不同 | 各自独立部署在不同硬件上 |
| 边界清晰 | 训练→模型 / Jetson→指令 / MCU→执行 |
| 独立迭代 | 各组件可各自升级，互不阻塞 |

### 2. 关键设计原则

#### 层次化抽象
```
┌────────────────────────────┐
│      应用层 (Training)      │  → AI模型训练
├────────────────────────────┤
│      决策层 (Jetson)        │  → 实时推理决策
├────────────────────────────┤
│      执行层 (MCU)           │  → 底层硬件控制
├────────────────────────────┤
│      物理层 (Go-Kart)       │  → 机械结构/执行器
└────────────────────────────┘
```

#### 协议驱动解耦
- MCU ⇄ Jetson: 串口自定义协议 (22字节帧)
- MCU ⇄ 伺服: CAN总线协议
- 模型格式: .pth checkpoint
- 配置: YAML文件

#### 渐进式模式设计
```
Manual → Remote → Training → Follow
  (手动)   (遥控)   (数据采集)  (自动驾驶)

每个模式增加一层能力:
  Manual: 直接驾驶
  +Remote: RC遥控控制
  +Training: 数据记录上传
  +Follow: AI自行驾驶
```

### 3. 安全设计

| 安全机制 | 实现 |
|----------|------|
| 帧校验 | CRC16-MODBUS防数据错误 |
| 超时保护 | 1秒无摄像头帧→强制刹车 |
| 刹车迟滞 | on/off计数防高频抖动 |
| 无人检测 | target_valid门控, <0.45强制停车 |
| 低通滤波 | steer_alpha=0.3防抖 |
| 限幅 | max_abs_steer=0.6, max_throttle=0.8 |
| 释放限制 | 松刹车后渐进恢复油门 |
