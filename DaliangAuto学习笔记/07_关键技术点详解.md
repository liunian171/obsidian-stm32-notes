# 关键技术点详解

## 一、STM32外设配置要点

### TIM输入捕获测量PWM
```c
// 以TIM2 CH1为例 (PA0)
// 配置要点:
// 1. 预分频器: 71 → 72MHz/(71+1) = 1MHz = 1us计数
// 2. 捕获模式: 上升沿触发, 直接捕获
// 3. 中断使能: CC1IE

// 中断处理:
void TIM2_IRQHandler(void) {
    if (TIM2->SR & TIM_SR_CC1IF) {
        uint32_t capture = TIM2->CCR1;  // 当前捕获值
        uint32_t period;
        if (capture > last_capture) {
            period = capture - last_capture;
        } else {
            period = (0xFFFF - last_capture) + capture + 1;
        }
        last_capture = capture;
        pulse_width = period;  // 单位: us
        // RC PWM: 1000~2000us 对应 0°~180°
    }
}
```

### CAN配置要点
```c
// 1Mbps CAN配置
CAN_HandleTypeDef hcan;
hcan.Init.Prescaler = 3;        // APB1=36MHz → 36M/3=12M
hcan.Init.SyncJumpWidth = CAN_SJW_1TQ;
hcan.Init.TimeSeg1 = CAN_BS1_9TQ;
hcan.Init.TimeSeg2 = CAN_BS2_2TQ;
// 时间量子: (1+9+2)=12TQ → 12M/12=1Mbps
hcan.Init.Mode = CAN_MODE_NORMAL;
hcan.Init.AutoBusOff = ENABLE;  // ABOM自动恢复
hcan.Init.AutoRetrans = ENABLE; // 自动重传
```

### 无RTOS的多任务调度模式
```
关键技巧:
1. 中断只做最轻量的工作 — 存数据到缓冲区,置标志位
2. 主循环做所有处理 — 解析/计算/控制
3. 每个循环都不阻塞 — 检查标志位,处理,继续
4. 使用计时变量实现周期性任务 — 如50ms CAN心跳
```

## 二、神经网络架构要点

### 为什么用ResNet18?
```
优点:
- 轻量: ~11M参数, 适合Jetson实时推理
- 预训练: ImageNet权重, 迁移学习效果好
- 残差连接: 解决梯度消失, 训练稳定
- 成熟: torchvision直接调用

选择考量:
- 比VGG轻量, 比MobileNet精度高
- Jetson Orin Nano可运行在30fps+
- 5帧×3推理=15次/秒, 足够
```

### 为什么用GRU?
```
相比LSTM:
- 参数量更少 (3个门 vs 4个门)
- 计算速度更快
- 小数据量下表现相近
- 适合Jetson上实时推理

GRU公式:
  z_t = σ(W_z · [h_{t-1}, x_t])      // 更新门
  r_t = σ(W_r · [h_{t-1}, x_t])      // 重置门
  h̃_t = tanh(W · [r_t * h_{t-1}, x_t]) // 候选隐藏
  h_t = (1-z_t) * h_{t-1} + z_t * h̃_t // 最终隐藏
```

### 为什么是5帧?
```
录制频率: 10fps → 5帧 = 0.5秒窗口
选择理由:
  2帧: 无法捕捉趋势
  3帧: 最小可感知识别
  5帧: 足够捕捉转向意图, 不过度增加计算
  10帧: 1秒窗口, 但GRU计算量大增
  结论: 5帧是实时性的最优平衡
```

## 三、UART协议设计要点

### 帧结构设计原则
```
1. SOF (帧头)     → 同步: 2字节0xAA55, 不易误判
2. VER (版本)     → 兼容: 未来协议升级
3. TYPE (类型)    → 多路: 不同消息类型
4. SEQ (序列号)   → 防丢: 检测丢帧/乱序
5. TS_MS (时间戳) → 同步: 用于数据对齐
6. DATA (数据)    → 有效载荷
7. CRC16 (校验)   → 可靠: 校验数据完整性

总长22字节, 其中有效数据12字节
效率: 12/22 = 54.5%
在115200bps下: 115200/(22×10) ≈ 523帧/秒
50Hz训练数据仅用 ~10%带宽
```

### CRC16-MODBUS vs 其他校验
```
CRC16: 多项式0x8005(标准) / 0xA001(反映射)
  检测能力: 所有单bit/双bit/奇数bit错误
  检测能力: 所有长度≤16的突发错误
  检测能力: 99.998%的更长发错误

简易校验和: 只能检测基本错误
MD5: 计算量太大, 不适合MCU
CRC16: 最适合嵌入式通信
```

## 四、Jetson多线程安全

### 线程间共享数据
```python
# 安全模式: threading.Lock + deque
self._lock = threading.Lock()
self._latest_pred = PredCommand(0, 0, 0)

# 生产者 (infer_loop)
with self._lock:
    self._latest_pred = new_pred

# 消费者 (send_loop)  
with self._lock:
    cmd = self._latest_pred
```

### 为什么不用Queue?
```
数据场景: 最新值覆盖旧值 (而非队列消费)
Queue适合: 1:1, 每条数据都要处理
Lock+变量适合: 只关心最新值, 丢弃旧值
推理20Hz输出, 发送20Hz → 不需要缓冲
```

## 五、数据采集与同步

### MCU端时间戳
```c
// 系统滴答计时器, 1ms分辨率
uint32_t tick_ms = HAL_GetTick();

// 训练数据帧格式
frame.ts_ms = tick_ms;         // 4字节时间戳
frame.speed = wheel_speed;     // 轮速
frame.steer = servo_position;  // 转向角度
frame.thr_ref = pedal_adc;     // 踏板ADC值
frame.brake = brake_switch;    // 刹车状态
frame.gear = gear_position;    // 档位

// 50Hz发送: 每20ms发送一次
static uint32_t last_send = 0;
if (tick_ms - last_send >= 20) {
    TrainUART_SendFrame(&frame);
    last_send = tick_ms;
}
```

### Jetson端时间对齐
```python
# 控制值插值: 获取精确时刻的控制值
def get_at_time(target_ts):
    # 在_history队列中查找target_ts前后两个帧
    c0, c1 = find_surrounding(target_ts)
    if not c0 or not c1:
        return None
    # 线性插值
    t = (target_ts - c0.ts_ms) / (c1.ts_ms - c0.ts_ms)
    steer = c0.steer + (c1.steer - c0.steer) * t
    throttle = c0.throttle + (c1.throttle - c0.throttle) * t
    return Control(steer, throttle)
```

## 六、安全后处理管线

```
模型输出 [steer, throttle, brake_logit, tv_logit]
    │
    ├── sigmoid(brake_logit) → brake_prob
    ├── sigmoid(tv_logit) → tv_prob
    │
    ├── tv_prob < 0.45? → 无人!
    │   steer=0, throttle=0, brake=1
    │
    ├── brake_prob > 0.65 (持续3帧)? → 刹车
    │   throttle=0, brake=1
    │
    ├── brake_prob < 0.35 (持续5帧)? → 松刹车
    │   brake=0, throttle渐进恢复
    │
    ├── steer低通: steer = α·steer + (1-α)·steer_raw
    │   steer_deadband: |steer| < 0.03 → 0
    │   steer_clamp: |steer| > 0.6 → 0.6
    │
    └── throttle_clamp: > 0.8 → 0.8
        throttle_release: 松刹车后5帧限制油门
```
