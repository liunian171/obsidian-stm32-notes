# MCU固件深度分析 — STM32F103RCT6

## 硬件平台

| 项目 | 规格 |
|------|------|
| MCU | STM32F103RCT6 (Cortex-M3 @72MHz) |
| Flash | 256KB |
| RAM | 48KB |
| 时钟 | HSE 8MHz晶振 → PLL×9 = 72MHz |
| 开发环境 | STM32CubeIDE (GCC) |
| HAL版本 | STM32Cube FW_F1 V1.8.6 |
| 无RTOS | Bare-metal 超级循环架构 |

## 外设使用分析

### 1. CAN — 转向伺服通信
```
引脚: PA11(CAN1_RX), PA12(CAN1_TX)
配置: Prescaler=3, BS1=9TQ, BS2=2TQ → 1Mbps
特性: ABOM(自动离线恢复), 自动重传
协议: 标准ID 0x64+node_id, 8字节数据帧
命令集:
  - Link(0x08), Enable(0x25), SetMaxSpeed(0x23)
  - GoToAbsolutePosition(0x22), MoveRelative(0x21)
  - StopHold(0x26), StopDisable(0x28)
  - ClearAlarm(0x81), WriteZero(0x50)
响应: Heartbeat(0x10)含32位绝对位置, Alarm(0x05)
```

### 2. USART — 3路串口
| 串口 | 引脚 | 波特率 | 用途 |
|------|------|--------|------|
| USART1 | PA9/PA10 | 115200 | Jetson训练/跟随数据链路 |
| USART2 | PA2/PA3 | 9600 | VGUS串口触摸屏 |
| USART3 | PB10/PB11 | 115200 | 预留 |
| UART4 | PC10/PC11 | 115200 | 预留 |
| UART5 | PC12/PD2 | 115200 | 预留 |

### 3. TIM — 5个定时器复用
```
TIM2: 输入捕获模式 — RC遥控器PWM信号解析 (CH1=转向, CH2=油门)
TIM3: PWM输出模式 — 电机驱动 (CH1/CH2, 1KHz)
TIM4: 输入捕获模式 — 轮速霍尔传感器脉冲计数 (CH3/CH4)
TIM8: 混合模式 — 输入捕获(轮速) + PWM输出(预留)
```

### 4. ADC — 模拟量采集
```
ADC1: 连续转换模式
  CH8(PB0) — 踏板霍尔传感器
  CH12(PC2) — 电池电压检测
  CH4(PA4), CH5(PA5), CH9(PB1) — 预留
```

### 5. I2C — OLED显示
```
I2C1: PB6(SCL), PB7(SDA), 100KHz
SSD1306 OLED 128×64, 地址0x3C
用于调试信息显示
```

## 软件架构

### 4种工作模式的状态机

```
MANUAL (0) ──→ REMOTE (1) ──→ TRAINING (2) ──→ FOLLOW (3)
   ↑              ↑                ↑                ↑
   │  踏板直驱    │  RC遥控控制    │  RC+数据上传   │  Jetson控制
   └──────────────┴────────────────┴────────────────┘
         全部可通过触摸屏切换，可随时返回
```

### 超级循环主逻辑（main.c）
```
while(1) {
    1. WheelSpeed_Task()         // 轮速计算
    2. Screen心跳/超时检测       // 触摸屏通信
    3. Screen按键读取 → 模式/档位/速度切换
    4. Follow模式: 轮询USART1指令
    5. 档位逻辑: Stop/Revert/Brake控制
    6. 物理档位开关消抖
    7. CAN伺服心跳 (50ms)
    8. CAN RX消息处理
    9. 电机PWM计算 (模式相关)
    10. 转向控制 (模式相关)
    11. 训练模式: 20ms数据上传
}
```

### 模块划分

| 模块 | 文件 | 职责 |
|------|------|------|
| 主控 | main.c | 初始化、主循环、模式切换 |
| RC输入 | rc_input.c | 读取RC接收机PWM信号 |
| CAN伺服 | servo_can.c | CAN总线伺服电机控制协议 |
| 电机控制 | motor_control.c | PWM电机驱动 |
| 轮速测量 | wheel_speed.c | 霍尔传感器脉冲计数→速度 |
| 屏幕 | screen.c | VGUS串口触摸屏协议 |
| 训练UART | train_uart.c | Jetson通信协议 |
| OLED | oled_ssd1306.c | I2C调试显示 |
| ADC | adc.c | 踏板/电池测量 |

## 通信协议

### 22字节训练/跟随帧
```
偏移  大小  字段
0-1    2    SOF: 0xAA 0x55
2      1    VER: 0x01
3      1    TYPE: 0x01训练/0x02跟随就绪/0x10主机控制
4-5    2    SEQ: 序列号(LE)
6-9    4    TS_MS: 时间戳(LE uint32)
10-11  2    SPD_MEAS: 速度0.1km/h(LE int16)
12-13  2    STEER: 转向0.1°(LE int16)
14-15  2    THR_REF: 油门0-1000(LE uint16)
16     1    BRAKE: 0/1
17     1    GEAR: 0=P/1=R/2=N/3=D
18-19  2    FLAGS: 预留
20-21  2    CRC16: MODBUS多项式0xA001
```

## 设计亮点与学习要点

1. **无RTOS的裸机架构** — 中断驱动+超级循环，适合资源受限MCU
2. **TIM输入捕获测量PWM** — 利用定时器硬件精确测量遥控器信号
3. **CAN伺服总线控制** — 标准化CAN协议控制伺服电机位置
4. **串口协议设计** — 完整的帧结构+CRC校验，工业级可靠性
5. **多模式状态机** — 清晰定义各模式行为，易于扩展
6. **档位消抖** — 50ms软件消抖+500ms启动忽略窗口
7. **训练数据时间同步** — MCU带时间戳上传，Jetson按时间戳对齐
8. **脚踏板ADC + RC双输入** — 支持手动/遥控双模式控制
