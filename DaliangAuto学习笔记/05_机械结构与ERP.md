## 一、铝型材卡丁车底盘

### 项目定位
大亮Auto设计的铝型材小型卡丁车平台，是整套视觉跟随系统的**物理载体**。

### 设计目标
- 紧凑尺寸：能放入大众ID.3后备箱
- DIY友好：铝型材框架，易于装配调试
- 开源非商业：CC BY-NC 4.0协议

### 硬件组成

```
┌──────────────────────────────────────────────┐
│              铝型材框架(900×180mm)             │
│  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐     │
│  │ 悬架│  │ 悬架│  │ 悬架│  │ 悬架│  │    │     │
│  │ 左前│  │ 右前│  │ 左后│  │ 右后│  │    │     │
│  └────┘  └────┘  └────┘  └────┘  └────┘     │
│         ║                            ║        │
│    [转向伺服] ←─ 拉杆 ──→ [前轮]     │        │
│                                              │
│    [轮毂电机L] ←── 后桥 ──→ [轮毂电机R]       │
│                                              │
│    [刹车电缸] ──→ 刹车线 ──→ [前轮线刹]      │
│                                              │
│    [STM32控制板]  [Jetson Orin Nano]          │
│    [IMX219摄像头]                             │
└──────────────────────────────────────────────┘
```

### 关键部件

| 部件 | 类型 | 说明 |
|------|------|------|
| 框架 | 铝型材(欧标) | 900×180mm，可拆装 |
| 悬架 | 4轮独立减震 | 弹簧避震 |
| 驱动 | 无刷轮毂电机×2 | 后轮直驱，各带霍尔传感器 |
| 转向 | 直流伺服电机 | CAN总线控制，带位置编码器 |
| 刹车 | 电动刹车缸 | 拉动刹车线，前轮线刹 |
| 计算 | Jetson Orin Nano | CSI接口+UART |
| 控制 | STM32F103RCT6 | 自研控制板 |
| 视觉 | IMX219 CSI摄像头 | 800万像素 |

### 物料清单
- 项目提供BOM v1.0和v1.1 (PDF)
- 3D模型：STEP格式整车装配体
- 设计图纸：型材框架900_180

## 二、Dliang-Open-ERP 进销存系统

### 技术栈

| 组件 | 技术 |
|------|------|
| Web框架 | FastAPI (Python 3.10+) |
| 数据库 | SQLite + SQLAlchemy ORM |
| 模板 | Jinja2 (服务端渲染) |
| 前端 | Bootstrap 5 + DataTables |
| 报表 | ReportLab (PDF), openpyxl (Excel) |
| 部署 | Docker + Docker Compose (+ Nginx) |

### 功能模块

```
Dliang-Open-ERP
├── 基础数据
│   ├── 物料管理 (Materials)
│   ├── BOM管理 (物料清单)
│   └── 客户/供应商管理
├── 采购管理
│   ├── 采购订单
│   └── 采购入库
├── 销售管理
│   ├── 销售订单
│   └── 销售出库
├── 库存管理
│   ├── 库存查询
│   ├── 库存盘点
│   └── 库存调拨
├── 生产管理
│   ├── 生产工单
│   └── 领料/退料
├── 财务管理
│   ├── 应收/应付
│   └── 收付款
└── 系统管理
    ├── 用户权限
    └── 操作日志
```

### 与DaliangAuto项目的关系
ERP系统可用于管理卡丁车项目的物料BOM、采购零件、库存跟踪等。

## 学习要点

### 机械设计方面
1. **铝型材框架设计** — 模块化、易调整、适合原型开发
2. **选型匹配** — 轮毂电机、伺服电机、电缸的选型逻辑
3. **3D模型管理** — STEP格式、BOM清单的重要性
4. **开源协议选择** — CC BY-NC 4.0保护非商业开源

### ERP设计方面
1. **FastAPI全栈应用** — 现代Python后端框架实践
2. **模块化设计** — 采购/销售/库存/生产各模块独立
3. **ORM数据建模** — SQLAlchemy多表关联设计
4. **服务端渲染** — Jinja2+Bootstrap的传统Web方案
5. **报表生成** — ReportLab PDF + openpyxl Excel导出
