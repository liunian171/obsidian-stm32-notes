# 通信协议详解

## 协议总览

系统涉及3种通信方式：

| 协议 | 物理层 | 速率 | 用途 |
|------|--------|------|------|
| UART-自定义帧 | USART1(PA9/PA10) | 115200bps | MCU ↔ Jetson |
| UART-VGUS | USART2(PA2/PA3) | 9600bps | MCU ↔ 触摸屏 |
| CAN | CAN1(PA11/PA12) | 1Mbps | MCU ↔ 转向伺服 |

## 一、UART自定义帧协议 (MCU ↔ Jetson)

### 帧结构 (22字节)

```
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│ SOF │ SOF │ VER │ TYPE│ SEQ │ SEQ │ TS_MS  (4B) │
│ 0xAA│ 0x55│0x01 │     │ (LE)│     │  LE uint32  │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┤
│  └────────┤  SPEED_01  │ STEER_01   │ THR_REF   │
│   TS_MS   │  int16 LE  │ int16 LE   │ uint16 LE │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┤
│ BRAKE│GEAR│ FLAGS      │ CRC16       │
│  1B  │ 1B │  uint16 LE │ uint16 LE   │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
```

### 帧类型

| TYPE | 名称 | 方向 | 频率 |
|------|------|------|------|
| 0x01 | 训练数据 | MCU→Jetson | 50Hz |
| 0x02 | 跟随就绪 | MCU→Jetson | 单次 |
| 0x05 | 手动通知 | MCU→Jetson | 单次 |
| 0x06 | 遥控通知 | MCU→Jetson | 单次 |
| 0x10 | 主机控制 | Jetson→MCU | 20Hz |

### CRC16-MODBUS实现
```c
// 多项式: 0xA001 (反映射0x8005)
uint16_t crc16_modbus(const uint8_t* data, uint16_t len) {
    uint16_t crc = 0xFFFF;
    for (uint16_t i = 0; i < len; i++) {
        crc ^= data[i];
        for (int j = 0; j < 8; j++) {
            if (crc & 1) crc = (crc >> 1) ^ 0xA001;
            else crc >>= 1;
        }
    }
    return crc;
}
```

### MCU端代码结构 (接收ISR)
```c
// USART1中断: 逐字节接收环形缓冲区
void USART1_IRQHandler(void) {
    uint8_t byte = USART1->DR;
    train_uart_rx_buf[train_uart_rx_tail] = byte;
    train_uart_rx_tail = (train_uart_rx_tail + 1) % BUF_SIZE;
}

// 主循环: 提取并校验帧
void TrainUART_PollFollowCmd(FollowCmd_t* cmd) {
    while (have_data_in_buffer()) {
        if (find_sof(0xAA, 0x55)) {
            uint16_t crc_calc = crc16_modbus(frame_data, 20);
            uint16_t crc_recv = (frame[21] << 8) | frame[20];
            if (crc_calc == crc_recv) {
                parse_frame(frame, cmd);
                return;
            }
        }
    }
}
```

### Jetson端解析
```python
def parse_frame_with_type(buf, max_steer_deg):
    # SOF检测: 滑动窗口搜索 0xAA 0x55
    idx = buf.find(b'\xaa\x55')
    if idx < 0 or idx + FRAME_LEN > len(buf):
        return None, None
    frame = buf[idx:idx+FRAME_LEN]
    # CRC校验
    if crc16_modbus(frame[:20]) != struct.unpack('<H', frame[20:22])[0]:
        return None, None  # CRC错误,丢帧
    # 解析字段
    type_ = frame[3]
    steer = struct.unpack('<h', frame[12:14])[0] / (max_steer_deg * 10)
    throttle = struct.unpack('<H', frame[14:16])[0] / 1000.0
    brake = bool(frame[16])
    gear = frame[17]
    speed = struct.unpack('<h', frame[10:12])[0] / 10.0
    return type_, Control(steer, throttle, brake, gear, speed, ...)
```

## 二、CAN协议 (MCU ↔ 转向伺服)

### 配置参数
```
波特率: 1Mbps (APB1=36MHz, Prescaler=3, BS1=9, BS2=2)
CAN ID: 0x64 (标准格式) + node_id (默认0)
数据长度: 8字节
ABOM: 开启 (自动离线恢复)
自动重传: 开启
```

### 命令集

| 命令码 | 名称 | 说明 |
|--------|------|------|
| 0x08 | Link | 连接检测 |
| 0x25 | Enable | 使能伺服 (参数: 0/1) |
| 0x23 | SetMaxSpeed | 设置最大速度 |
| 0x22 | GoToAbsolutePosition | 绝对位置控制 (4字节int32位置) |
| 0x21 | MoveRelative | 相对位置移动 |
| 0x20 | SetSpeedMode | 速度模式 |
| 0x26 | StopHold | 停止保持 |
| 0x28 | StopDisable | 停止释放 |
| 0x81 | ClearAlarm | 清除报警 |
| 0x50 | WriteZero | 写零点 |

### 伺服心跳响应 (0x10)
```
8字节数据: [0x10, status, position_LE(4B), speed_LE(2B)]
position: 32位绝对位置编码器值
```

## 三、VGUS触摸屏协议

```
帧格式: 0xA5 0x5A + LEN + CMD + ADDR(2B) + DATA(n)
写变量: CMD=0x82
读变量: CMD=0x83

变量地址:
  0x05E1 — 档位显示
  0x05C1 — 速度显示
  0x05A1 — 模式显示
  0x05E6 — 连接状态
  0x02E5 — 速度数值
  0x05EB — 电量显示
```

## 学习要点

1. **帧结构设计原则**: SOF + 版本 + 类型 + 数据 + CRC，工业标准做法
2. **CRC16校验**: MODBUS多项式(0xA001)，计算简单、检错能力强
3. **滑动窗口SOF搜索**: 鲁棒的帧同步机制，应对字节错位
4. **定时器+输入捕获**: 硬件精确测量PWM脉冲宽度
5. **CAN总线协议**: 标准化命令+心跳机制，适合工业伺服控制
6. **归一化映射**: MCU和Jetson之间的数据标准转换
7. **波特率选择**: 9600(屏幕, 简单数据) / 115200(主链路) / 1Mbps(CAN)
