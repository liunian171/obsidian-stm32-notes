# 基变换

## 核心思想

**不同的坐标系描述同一个向量，坐标不同，但箭头指向同一个位置。**

```
        标准基视角        vs        自定义基视角
        î = (1,0)                 u = (1,1)
        ĵ = (0,1)                 v = (1,-1)
        
        向量 (3,2)                同一个向量，别的坐标
```

## 基变换矩阵

假设我们有一组新基，用**相对于标准基**的坐标表示：

```
新基 B = [ u  v ]  其中 u = (1, 1), v = (-1, 1)
```

那么：**基变换矩阵就是把这些基向量作为列拼起来**。

```
M = [ 1  -1 ]
    [ 1   1 ]
```

**作用**：把"在 B 基下的坐标"转换为"在标准基下的坐标"。

```
[标准基坐标] = M · [B 基坐标]
```

## 反过来转换

要从标准基转换到 B 基：

```
[B 基坐标] = M⁻¹ · [标准基坐标]
```

> [!tip] 
> **M 就像"翻译器"，把 B 语言翻译成标准语言；M⁻¹ 是反向翻译。**

## 实际应用

### 1. 不同坐标系间的变换

```
     B 基下            标准基下           B 基下
     向量    ── M ──→   向量   ── 变换 ──→  结果
     坐标               坐标             坐标
      ↑                  ↑                ↑
      │                  │                │
    原始              在标准系中       在 B 系中
    输入              做变换           读取结果
```

完整流程：

```
结果(B 坐标) = M⁻¹ · A · M · 输入(B 坐标)
```

这叫做**相似变换**：`A' = M⁻¹AM`

### 2. 相似矩阵

如果两个矩阵满足 `B = P⁻¹AP`，则它们代表**同一个线性变换在不同基下的形式**。

相似矩阵共享：
- 相同的特征值
- 相同的行列式
- 相同的迹
- 相同的秩

## 类比

> 想象你跟一群外国人描述同一座建筑——你用"米"，他们用"英尺"。
> 数字不同，但建筑没变。基变换矩阵就是米和英尺之间的换算表。
